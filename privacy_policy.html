Umgang mit Google-Nutzerdaten

Diese Anwendung greift auf Daten der Google Calendar API zu, um ihre Kernfunktionalität zu ermöglichen. Die Nutzung von Informationen, die von Google APIs empfangen werden, erfolgt in Übereinstimmung mit der Google API Services User Data Policy, einschließlich der Anforderungen zur eingeschränkten Nutzung (Limited Use requirements).

Welche Daten werden verwendet und warum?

Unsere Anwendung fordert folgende Berechtigungen (Scopes) für Ihr Google-Konto an:

  • .../auth/calendar.readonly: Diese Berechtigung wird benötigt, um eine Liste Ihrer verfügbaren Google Kalender abzurufen. Dies ermöglicht es Ihnen, in der Anwendung den Ziel-Kalender auszuwählen, in den neue Termine eingetragen werden sollen. Wir lesen keine Termininhalte oder andere Details Ihrer Kalender.
  • .../auth/calendar.events: Diese Berechtigung wird ausschließlich dazu verwendet, neue Termine in dem von Ihnen ausgewählten Kalender zu erstellen. Die Inhalte der Termine (Titel, Beschreibung, Datum) werden lokal in der Anwendung basierend auf Ihrer Eingabe und der Konfigurationsdatei generiert. Es werden keine bestehenden Termine gelesen, geändert oder gelöscht.

Wie werden die Daten gespeichert?

Die Anwendung speichert keine Ihrer Kalenderdaten auf externen Servern. Die Authentifizierungsinformationen (ein sogenannter "Refresh Token") werden sicher und verschlüsselt im lokalen Anwendungsdatenverzeichnis auf Ihrem eigenen Computer gespeichert, um eine erneute Anmeldung bei jedem Start zu vermeiden. Diese Daten verlassen niemals Ihren Computer.

Keine Weitergabe von Daten

Ihre Google-Nutzerdaten werden unter keinen Umständen an Dritte weitergegeben oder für andere Zwecke (z.B. Werbung) verwendet.
